# Multi-Energy Optimization Model

This project implements a modular optimization framework using Pyomo for multi-energy systems. It supports various resource types (e.g., PV, batteries, CHP, hydrogen storage) and integrates electricity, gas, and reserves markets with energy balance constraints.

---

## 📁 Project Structure

- `main.py` — Entry point for running the full optimization pipeline.
- `config.py` — Centralized configuration: solver settings, paths, required fields.
- `read_inputs.py` — Parses and validates Excel input data.
- `initialize_resources.py` — Converts data dictionaries into Python resource objects.
- `run_optimization.py` — Coordinates the optimization steps.
- `optimization_model.py` — Defines the Pyomo model, builds and solves it.
- `bidding.py` — Adds bidding variables, constraints, and objective.
- `energy_balance.py` — Adds thermal and hydrogen energy balance constraints.
- `save_results.py` — Exports results to Excel files (debugging and structured).
- `resources_models/` — Directory containing all resource-specific models:
  - `resources.py` — Common base for resources models
  - `resources_PV.py` — PV generation
  - `resources_windgenerator.py` — Wind generation with wind curve
  - `resources_battery.py` — Battery system with SOC tracking
  - `resources_CHP.py` — Combined Heat and Power plant
  - `resources_electrolyzer.py` — Hydrogen production
  - `resources_fuelcell.py` — Hydrogen-to-electricity conversion
  - `resources_hydrogenstorage.py` — Hydrogen tank model
  - `loads.py` — Common base for loads models
  - `loads_electrical.py` — Electrical loads
  - `loads_thermal.py` — Thermal loads
  - `loads_hydrogen.py` — Hydrogen loads

- `helper/`
  - `helper.py` — Logging utility.
  - `exceptions.py` — Custom exception classes.

- `data_inputs/` — Folder for Excel input files.
- `data_outputs/` — Folder for results and output files.

---

## ✅ Features

- Object-oriented structure for adding and managing energy resources.
- Electricity and gas market bidding with reserve participation.
- Support for thermal and hydrogen energy balances.
- Debugging and structured Excel exports of model outputs.
- Easy extensibility for new resource types or energy carriers.
- Configurable solver (`cplex`, `glpk`, `highs`, etc.).

## ⚙️ How to Install

### 1. Installation

You can clone and run the project directly from the repository:

```bash
git clone https://gitlab.inesctec.pt/amcoelho/crete_valley_multi_energy_tool.git
cd crete_valley_multi_energy_tool
```

### 2. Set up the environment

It is recommended to use a virtual environment:

```bash
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

### 3. Install dependencies

Install the required Python packages:

```bash
pip install -r requirements.txt
```

### 4. Install a solver

- CPLEX, HiGhS, GLPK, etc

## ⚙️ How to Use

### 1. Prepare Input File
Place your Excel file inside `data_inputs/` named according to `CELL_NAME` in `config.py`, e.g.: data_inputs/data_inputs_cell_1.xlsx

Expected sheets:
- `"Energy resources"` — Parameters for each resource.
- `"PV generation"`, `"Wind generation"`, `"Electrical loads"`, etc. — Time series profiles.

### 2. Configure Settings
In `config.py`:
- Set the bidding time, solver, and system reserve flags.
- Ensure `REQUIRED_FIELDS_BY_TYPE` and `REQUIRED_LABELS_PRICES` match your data.

### 3. Run the Optimization
```bash
python main.py
```

## 4. View Results

Results will be saved in the `data_outputs/` folder, including:

- `results_with_reserves.xlsx`
- `results_with_reserves_debugging.xlsx`

Or the "no reserves" variants, depending on the configuration in `config.py`.

## 📤 Output Files

### Debugging Files

Generated by `save_model_variables_to_excel_debugging`:

- One sheet with **all model variables**
- Additional sheets filtered by:
  - `PV`
  - `WG` (wind generator)
  - `CHP`
  - `sto_E` (batteries)
  - `EL` (electrolyzers)
  - `FC` (fuel cells)
  - `sto_H2` (hydrogen storage)
  - `load` (all loads)

### Structured Files

Generated by `save_model_to_excel`, using each resource’s `save_results()` method.

## 🧩 Adding a New Resource

1. Create a new class in `resources_models/`  
   _(e.g., `resources_heatpump.py`)_

2. Implement the following methods:
   - `get_variables(model)`
   - `get_constraints(model)`
   - `save_results(model, writer)`

3. Update `initialize_resources.py` to handle the new resource type.

4. Update `save_results.py` to handle the new resource type.

(Optional)

5. Update `read_inputs.py` if the resource has a profile.

---

## 🚫 Known Limitations

- All resources must share the same `system_reserves_participation` setting.
- The model assumes one profile per time step per resource.
- Excel naming patterns (e.g., `"sto_H2"`, `"EL"`) are used for filtering — avoid conflicts.
- It only considers electricity energy and secondary reserves, and natural gas bidding.
- It only considers electricity, gas, thermal, and hydrogen vectors.
